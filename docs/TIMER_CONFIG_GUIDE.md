## 🎯 推荐方案

### 方案一：TIM2（当前实现）
**最佳选择：通用定时 + 备用通道**

✅ **优点：**
- 32位计数器，最大延时时间长
- 4个输出通道可用于PWM
- 中断 + PWM双用途
- 适合需要灵活配置的场景

❌ **缺点：**
- 占用较多资源（如果只做定时）

**配置参数：**
```c
定时器：TIM2
预分频：95（PSC）
重载值：999（ARR）
中断周期：1ms
```

---

### 方案二：TIM6（推荐用于纯定时）
**最简洁：专用基本定时器**

✅ **优点：**
- 专为定时设计，资源占用最小
- 不占用输出通道
- 代码最简洁
- 适合只需要定时中断的场景

❌ **缺点：**
- 无PWM功能
- 16位计数器（但1ms应用足够）

**配置参数：**
```c
定时器：TIM6
预分频：95（PSC）
重载值：999（ARR）
中断周期：1ms
```

**切换方法：** 修改 `timer_config.h` 中的宏定义：
```c
// #define USE_TIM2      // 注释掉
#define USE_TIM6         // 启用TIM6

#ifdef USE_TIM6
    #define TIMx                    TIM6
    #define TIMx_IRQn               TIM6_IRQn
    #define TIMx_IRQHandler         TIM6_IRQHandler
    #define RCC_APBxPeriph_TIMx     RCC_APB1Periph_TIM6
    #define RCC_APBxPeriphClockCmd  RCC_APB1PeriphClockCmd
#endif
```

## 📐 常用中断周期配置速查表

### 系统时钟：96MHz，定时器时钟：96MHz

| 目标周期 | 频率 | 预分频(PSC) | 重载值(ARR) | 计算验证 |
|---------|------|------------|------------|---------|
| **1ms** | 1kHz | 95 | 999 | 96M/(96×1000)=1kHz ✓ |
| **10ms** | 100Hz | 95 | 9999 | 96M/(96×10000)=100Hz ✓ |
| **100ms** | 10Hz | 959 | 9999 | 96M/(960×10000)=10Hz ✓ |
| **500μs** | 2kHz | 47 | 999 | 96M/(48×1000)=2kHz ✓ |
| **100μs** | 10kHz | 95 | 99 | 96M/(96×100)=10kHz ✓ |
| **10μs** | 100kHz | 9 | 95 | 96M/(10×96)=100kHz ✓ |
| **1μs** | 1MHz | 95 | 0 | 96M/(96×1)=1MHz ✓ |

### 计算公式

```
中断频率 = 定时器时钟 / [(PSC+1) × (ARR+1)]

例如1ms中断：
1000Hz = 96,000,000Hz / [(95+1) × (999+1)]
1000Hz = 96,000,000Hz / [96 × 1000]
1000Hz = 96,000,000Hz / 96,000 ✓
```

## ⚙️ 精度分析

### 1ms定时精度

**理论精度：**
```
系统时钟：96MHz
时钟周期：1/96MHz ≈ 10.4ns
1ms理论计数：96,000个时钟周期
```

**实际误差：**
```
配置：PSC=95, ARR=999
实际周期 = 96,000,000 / (96 × 1000) = 1000.00 Hz
误差：0%（完美匹配）
```

**精度影响因素：**
1. 晶振精度（±50ppm 典型值）
2. 温度漂移（±20ppm/℃）
3. 中断响应延迟（< 1μs）

**长时间运行误差估算：**
```
假设晶振精度±50ppm：
1小时误差：3600s × 50ppm = 0.18s
1天误差：86400s × 50ppm = 4.32s
```

## 🎛️ 中断优先级配置建议

### NVIC优先级分组（当前：Group 2）

```c
NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);
// 2位抢占优先级，2位子优先级
// 抢占优先级：0-3（0最高）
// 子优先级：0-3（0最高）
```

### 不同场景的优先级建议

| 应用场景 | 抢占优先级 | 子优先级 | 说明 |
|---------|----------|---------|------|
| **通用定时** | 2 | 0 | 默认中等优先级 |
| **电机控制** | 0 | 0 | 最高优先级，不能被打断 |
| **通信协议** | 1 | 0 | 高优先级，避免丢包 |
| **按键扫描** | 3 | 0 | 低优先级 |
| **LED显示** | 3 | 1 | 最低优先级 |

### 优先级配置示例

```c
// 示例1：高优先级定时器（电机PWM）
NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 0;
NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;

// 示例2：中等优先级（1ms系统心跳）
NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 2;
NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;

// 示例3：低优先级（GUI刷新）
NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 3;
NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;
```

## 📝 切换定时器步骤总结

### 从TIM2切换到TIM6的完整步骤

1. **修改 `timer_config.h`**
   ```c
   // 注释掉USE_TIM2
   // #define USE_TIM2

   // 启用USE_TIM6
   #define USE_TIM6

   #ifdef USE_TIM6
       #define TIMx                    TIM6
       #define TIMx_IRQn               TIM6_IRQn
       #define TIMx_IRQHandler         TIM6_IRQHandler
       #define RCC_APBxPeriph_TIMx     RCC_APB1Periph_TIM6
       #define RCC_APBxPeriphClockCmd  RCC_APB1PeriphClockCmd

   ```

2. **无需修改其他代码**
   - `timer_config.c` 自动适配

## 📌 最佳实践建议

1. ✅ **推荐TIM6用于纯定时功能**（资源占用最小）
2. ✅ **推荐TIM2用于定时+PWM复合需求**（功能丰富）
3. ✅ **推荐TIM5用于长时间计数**（32位计数器）
4. ⚠️ **避免在ISR中执行耗时操作**（保持< 10μs）
5. ⚠️ **避免在ISR中使用printf**（可能导致死锁）
6. ⚠️ **避免在ISR中调用延时函数**（破坏定时精度）
