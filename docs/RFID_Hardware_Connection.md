## 🔧 接口说明

### RFID模块接口定义

RFID读写器模块一般有以下接口：

| 引脚序号 | 引脚名称 | 功能说明 | 电气特性 |
|---------|---------|---------|---------|
| 1 | VCC | 电源正极 | DC 5V |
| 2 | GND | 电源地 | GND |
| 3 | TXD | 串口发送 | TTL 3.3V输出 |
| 4 | RXD | 串口接收 | TTL 3.3V输入 |
| 5 | LED（可选） | 指示灯 | 读卡时点亮 |
| 6 | BEEP（可选） | 蜂鸣器 | 读卡时鸣叫 |

**注意：**
- TXD/RXD为TTL 3.3V电平，可直接连接CH32V30x
- 部分模块LED/BEEP引脚可能没有引出
- 电源必须使用5V，模块内部有3.3V转换

## 📍 引脚连接

### 方案1：使用UART5（推荐）

UART5是CH32V30x的专用UART接口，不占用USART资源。

| RFID模块 | CH32V30x | 引脚功能 | 说明 |
|---------|---------|---------|------|
| VCC (5V) | 5V | 电源 | 由外部5V电源供电 |
| GND | GND | 地 | 公共地 |
| TXD | PC12 | UART5_RX | RFID模块发送→单片机接收 |
| RXD | PD2 | UART5_TX | RFID模块接收←单片机发送 |

**引脚复用检查：**
- PC12：UART5_RX（本项目使用）
- PD2：UART5_TX（本项目使用）

**电路连接：**
```
[RFID模块]          [CH32V307VCT6]
  VCC (5V) ---------> 5V电源
  GND ---------------> GND
  TXD ---------------> PC12 (UART5_RX)
  RXD <--------------- PD2  (UART5_TX)
```

### 方案2：使用其他UART（备选）

如果UART5被占用，可以选择其他UART接口：

#### UART6方案

| RFID模块 | CH32V30x | 引脚功能 |
|---------|---------|---------|
| TXD | PC0 | UART6_RX |
| RXD | PC1 | UART6_TX |

#### UART7方案

| RFID模块 | CH32V30x | 引脚功能 |
|---------|---------|---------|
| TXD | PC2 | UART7_RX |
| RXD | PC3 | UART7_TX |

#### USART3方案（如需重映射）

| RFID模块 | CH32V30x | 引脚功能 |
|---------|---------|---------|
| TXD | PD9 | USART3_RX (重映射) |
| RXD | PD8 | USART3_TX (重映射) |

## ⚡ 电气特性

### 电源要求

**RFID模块电源：**
- 输入电压：DC 5V ±5%
- 输入电流：待机<100mA，读卡<200mA
- 建议使用独立的5V稳压电源供电

**CH32V30x供电：**
- VDD：3.3V
- IO电平：3.3V TTL
- 5V容忍IO：部分IO支持5V输入（需查数据手册）

### 电平匹配

**RFID模块 → CH32V30x（RXD方向）：**
- RFID模块输出：3.3V TTL
- CH32V30x输入：3.3V TTL兼容
- ✅ 可以直接连接

**CH32V30x → RFID模块（TXD方向）：**
- CH32V30x输出：3.3V TTL
- RFID模块输入：3.3V TTL
- ✅ 可以直接连接

### 信号完整性

**推荐做法：**
1. TXD/RXD走线尽量短（<10cm）
2. 如果走线较长（>20cm），建议串联22Ω-47Ω电阻
3. GND就近连接，保证良好接地

**抗干扰措施：**
```
         22Ω              22Ω
PC12 ----/\/\/\---- TXD  PD2 ----/\/\/\---- RXD
(CH32)              (RFID)       (CH32)              (RFID)
```

### 电源设计建议

**方案A：独立5V电源供电（推荐）**
```
                      ┌─────────┐
外部电源 ──→ LDO ──→  │ 5V输出   │ ──→ RFID模块VCC
(7-12V)    (7805)     └─────────┘
                           │
                           └──→ 100μF电容 || 0.1μF电容
```

**方案B：USB 5V供电**
```
USB 5V ──→ 二极管 ──→ RFID模块VCC
             │
             └──→ 100μF电容 || 0.1μF电容
```

**滤波电容：**
- 靠近RFID模块VCC引脚放置100μF电解电容（抑制低频纹波）
- 靠近VCC引脚放置0.1μF陶瓷电容（抑制高频噪声）

## 🧪 硬件测试

### 步骤1：电源测试

**测试工具：** 万用表

**测试步骤：**
1. 断开RFID模块连接
2. 测量5V电源输出电压：应为**4.75V~5.25V**
3. 测量CH32V30x的3.3V：应为**3.2V~3.4V**
4. 确认GND正确连接

**判断标准：**
- ✅ 5V电源在4.75V~5.25V范围内
- ✅ 3.3V电源在3.2V~3.4V范围内
- ❌ 如电压异常，检查电源电路

### 步骤2：UART回环测试

**测试目的：** 验证UART硬件功能正常

**测试方法：**
1. 将PC12和PD2短接（UART5 RX和TX短接）
2. 编译并运行以下测试代码：

```c
void UART_LoopbackTest(void)
{
    uint8_t test_data[] = "UART5 Test";
    uint8_t rx_buf[32];

    // 发送测试数据
    USART_SendData(UART5, test_data, sizeof(test_data));
    Delay_Ms(100);

    // 接收数据
    if (USART_GetRxCount(UART5) > 0) {
        USART_ReadData(UART5, rx_buf, sizeof(test_data));

        // 比较数据
        if (memcmp(test_data, rx_buf, sizeof(test_data)) == 0) {
            printf("UART回环测试：通过 ✅\n");
        } else {
            printf("UART回环测试：失败 ❌\n");
        }
    }
}
```

**判断标准：**
- ✅ 收到的数据与发送的数据完全一致
- ❌ 如测试失败，检查UART初始化代码

### 步骤3：RFID模块连接测试

**测试目的：** 验证RFID模块硬件连接正确

**测试方法：**
1. 连接RFID模块到CH32V30x
2. 上电后观察RFID模块状态：
   - LED应该点亮或闪烁
   - 蜂鸣器可能会短鸣一声（部分模块）
3. 运行驱动初始化代码：

```c
void RFID_ConnectionTest(void)
{
    uint8_t version;
    RFID_Result ret;

    // 初始化RFID驱动
    RFID_Init(RFID_BAUDRATE);
    Delay_Ms(100);

    // 查询固件版本
    ret = RFID_QueryVersion(&version, 1000);
    if (ret == RFID_RET_OK) {
        printf("RFID连接测试：通过 ✅\n");
        printf("固件版本：V%d.%d\n", version >> 4, version & 0x0F);
    } else {
        printf("RFID连接测试：失败 ❌\n");
        printf("请检查：\n");
        printf("1. 电源连接是否正确\n");
        printf("2. TXD/RXD是否接反\n");
        printf("3. 波特率是否匹配（默认9600）\n");
    }
}
```

**判断标准：**
- ✅ 成功读取固件版本号
- ❌ 如读取失败，按提示检查连接

### 步骤4：读卡功能测试

**测试目的：** 验证RFID读卡功能正常

**测试方法：**
```c
void RFID_ReadCardTest(void)
{
    uint8_t card_type[2], uid[4];
    RFID_Result ret;

    printf("请将IC卡靠近读卡器...\n");

    // 命令模式读卡
    ret = RFID_ReadId(card_type, uid, 3000);
    if (ret == RFID_RET_OK) {
        printf("读卡测试：通过 ✅\n");

    } else {
        printf("读卡测试：失败 ❌\n");
        printf("错误代码：%d\n", ret);
        printf("可能原因：\n");
        printf("1. 卡片未靠近读卡器\n");
        printf("2. 卡片类型不支持\n");
        printf("3. 读卡器天线故障\n");
    }
}
```

**判断标准：**
- ✅ 成功读取IC卡号
- ❌ 如读取失败，检查卡片类型和距离

## ❓ 常见问题

### Q1: RFID模块无响应

**症状：** 查询版本号失败，无法通信

**排查步骤：**
1. **检查电源：**
   - 用万用表测量RFID模块VCC引脚，应为5V±0.25V
   - 检查电源容量是否足够（至少200mA）

2. **检查TXD/RXD连接：**
   ```
   正确连接：
   RFID_TXD → CH32_RX (PC12)
   RFID_RXD ← CH32_TX (PD2)

   如果接反会导致无法通信！
   ```

3. **检查波特率：**
   - RFID模块默认波特率为9600
   - 确认代码中`RFID_BAUDRATE`宏定义为9600

4. **检查GND连接：**
   - 必须共地，否则电平不稳定

### Q2: 读卡距离太近

**症状：** 必须贴紧才能读到卡

**可能原因：**
1. **电源电压不足：**
   - 测量5V电源，如果低于4.8V会影响发射功率
   - 解决方法：使用更大容量的5V电源

2. **天线干扰：**
   - RFID模块附近不要放金属物体
   - 保持天线与PCB板距离>5mm

3. **模块质量问题：**
   - 更换质量更好的RFID模块

### Q3: 读卡数据错误

**症状：** 能读到卡但数据不对

**可能原因：**
1. **信号干扰：**
   - TXD/RXD走线太长或靠近强干扰源
   - 解决方法：串联22Ω电阻，减少反射

2. **波特率不匹配：**
   - 检查RFID模块实际波特率设置
   - 使用`RFID_QueryWorkMode()`查询

3. **电源纹波大：**
   - 增加滤波电容（100μF + 0.1μF）

### Q4: 自动模式不工作

**症状：** 切换到自动模式后收不到数据

**排查步骤：**
1. **确认模式切换成功：**
   ```c
   RFID_SetWorkMode(RFID_MODE_AUTO_ID, 0, 1000);
   Delay_Ms(2000);  // 必须等待重启！

   // 查询确认
   RFID_WorkMode mode;
   RFID_QueryWorkMode(&mode, NULL, 1000);
   printf("当前模式: %d\n", mode);  // 应为2
   ```

2. **检查接收代码：**
   ```c
   RFID_Frame frame;
   // 使用自动帧接收函数
   RFID_WaitAutoFrame(&frame, 5000);  // 不是RFID_ReadId!
   ```

3. **检查帧类型：**
   - 自动帧的帧头是0x04，不是0x02

### Q5: 多卡同时靠近时冲突

**症状：** 两张卡同时靠近时读不出来

**解释：**
- ISO14443A协议支持防冲突，但需要读卡器固件支持
- XH3650系列模块通常只能识别一张卡

**解决方法：**
1. 提示用户一次只刷一张卡
2. 使用防冲突算法（需模块支持）
3. 使用UHF RFID（支持多标签读取）

### Q6: 5V电源和3.3V IO电平兼容性

**问题：** RFID模块用5V供电，但IO是3.3V，安全吗？

**回答：**
- ✅ **安全**。RFID模块内部有5V→3.3V转换
- TXD/RXD是3.3V TTL电平，可直接连接CH32V30x
- 不需要电平转换芯片
