## 1. 变更概述

### 1.1 实施目标

基于现有门禁控制系统，实现完整的用户认证功能：

- **用户数据库**: 支持50用户，Flash持久化存储
- **开门日志**: 256条循环记录，支持断电恢复
- **多认证方式**: RFID卡、指纹、4位密码
- **管理功能**: 主密码保护的用户管理界面

### 1.2 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 用户容量 | 50 | 约1.6KB Flash空间 |
| 日志容量 | 256条 | 约4KB，循环缓冲 |
| 密码格式 | 4位数字 | 键1-9=1-9, 键10=0 |
| 默认主密码 | 0000 | 可修改 |
| 开门时长 | 5秒 | 认证成功后自动锁门 |
| 锁定阈值 | 5次失败 | 锁定60秒 |

### 3.7 auth_manager.c (v1.1.0 补丁)

**路径**: `User/auth_manager.c`

**变更内容**: 主密码验证成功后进入管理模式

**问题**: 原实现中 `auth_process_password()` 验证主密码成功后仅返回成功，未调用 `user_admin_enter()` 导致无法进入管理模式。

```c
// 修改前:
if (user_db_verify_master_password(password) == USER_DB_OK) {
    printf("[AuthMgr] Master password verified\r\n");
    s_auth_mgr.state = AUTH_MGR_IDLE;
    return AUTH_RESULT_SUCCESS;
}

// 修改后:
if (user_db_verify_master_password(password) == USER_DB_OK) {
    printf("[AuthMgr] Master password verified, entering admin mode\r\n");
    user_admin_enter(password);   // 新增: 进入管理模式
    s_auth_mgr.state = AUTH_MGR_IDLE;
    return AUTH_RESULT_SUCCESS;
}
```

case ADMIN_STATE_UNLOCKED:
    switch (key_id) {
        case 1:  user_admin_start_add_user();         break;  // 添加用户
        case 2:  user_admin_start_delete_user();       break;  // 删除用户

        case 12:
        case 15: user_admin_exit();                    break;  // 退出
    }
    break;

case ADMIN_STATE_DELETE_USER:
    // 键1-9选择用户序号，通过user_db_get_by_index()获取
    if (key_id >= 1 && key_id <= 9) {
        user_entry_t user;
        if (user_db_get_by_index(key_id - 1, &user) == USER_DB_OK) {
            s_admin.selected_user_id = user.id;
        }
    }
    break;

case ADMIN_STATE_LIST_USERS:

## 4. 模块详细说明

### 4.1 用户数据库模块 (user_database)

**职责**: 管理用户信息的Flash持久化存储

**数据结构**:

```c
// 用户条目 (32字节)
typedef struct {
    uint16_t id;                // 用户ID (2字节)
    char name[12];              // 用户名 (12字节)
    uint8_t rfid_uid[4];        // RFID卡号 (4字节)
    uint16_t fingerprint_id;    // 指纹模板ID (2字节)
    uint8_t password[4];        // 4位密码 (4字节)
    uint16_t flags;             // 用户标志 (2字节)
    uint8_t reserved[6];        // 保留 (6字节)
} user_entry_t;

// 用户标志位
#define USER_FLAG_VALID     0x0001  // 用户有效
#define USER_FLAG_ADMIN     0x0002  // 管理员
#define USER_FLAG_RFID_EN   0x0004  // RFID已录入
#define USER_FLAG_FP_EN     0x0008  // 指纹已录入
#define USER_FLAG_PWD_EN    0x0010  // 密码已设置
#define USER_FLAG_DISABLED  0x0020  // 用户禁用
```

**核心函数**:

| 函数 | 说明 |
|------|------|
| `user_db_init()` | 初始化，检查Flash有效性 |
| `user_db_format()` | 格式化数据库 |
| `user_db_add_user()` | 添加用户 |
| `user_db_delete_user()` | 删除用户 |
| `user_db_find_by_rfid()` | 通过RFID查找 |
| `user_db_find_by_fingerprint()` | 通过指纹ID查找 |
| `user_db_verify_password()` | 验证用户密码 |
| `user_db_verify_master_password()` | 验证主密码 |

### 4.2 开门日志模块 (access_log)

**职责**: 记录所有认证尝试，支持循环覆盖

**数据结构**:

```c
// 日志记录 (16字节)
typedef struct {
    uint32_t timestamp;     // 时间戳/毫秒计数 (4字节)
    uint16_t user_id;       // 用户ID (2字节)
    uint8_t auth_method;    // 认证方式 (1字节)
    uint8_t result;         // 认证结果 (1字节)
    uint8_t rfid_uid[4];    // RFID卡号 (4字节)
    uint8_t reserved[4];    // 保留 (4字节)
} access_record_t;

// 认证结果枚举
typedef enum {
    AUTH_RESULT_SUCCESS = 0,
    AUTH_RESULT_FAILED,
    AUTH_RESULT_USER_NOT_FOUND,
    AUTH_RESULT_USER_DISABLED,
    AUTH_RESULT_TIMEOUT,
    AUTH_RESULT_DOOR_FORCED
} auth_result_t;
```

**核心函数**:

| 函数 | 说明 |
|------|------|
| `access_log_init()` | 初始化日志模块 |
| `access_log_record()` | 添加一条记录 |
| `access_log_get_count()` | 获取记录数 |
| `access_log_get_record()` | 获取指定记录 |
| `access_log_get_latest()` | 获取最新记录 |
| `access_log_clear()` | 清空日志 |

### 4.3 密码输入模块 (password_input)

**职责**: 处理4位数字密码的输入流程

**状态机**:

```
PWD_STATE_IDLE → PWD_STATE_INPUTTING → PWD_STATE_COMPLETE
                        ↓
                 PWD_STATE_TIMEOUT / PWD_STATE_CANCELLED
```

**核心函数**:

| 函数 | 说明 |
|------|------|
| `pwd_input_start()` | 开始密码输入 |
| `pwd_input_on_key()` | 处理按键输入 |
| `pwd_input_cancel()` | 取消输入 |
| `pwd_input_update()` | 检查超时 |
| `pwd_input_is_active()` | 检查是否在输入中 |

### 4.4 认证管理器 (auth_manager)

**职责**: 统一处理所有认证请求，管理认证状态

**状态机**:

```
AUTH_MGR_IDLE → AUTH_MGR_PROCESSING → 成功/失败
      ↓
AUTH_MGR_PASSWORD_INPUT → 密码完成 → 认证
      ↓
AUTH_MGR_LOCKED (失败5次) → 60秒后自动解锁
```

**核心函数**:

| 函数 | 说明 |
|------|------|
| `auth_manager_init()` | 初始化 |
| `auth_process_rfid()` | 处理RFID认证 |
| `auth_process_fingerprint()` | 处理指纹认证 |
| `auth_start_password()` | 启动密码输入 |
| `auth_process_password()` | 处理密码认证 |
| `auth_manager_is_locked()` | 检查锁定状态 |

### 4.5 用户管理模块 (user_admin)

**职责**: 提供用户管理界面（需主密码验证）

**状态机**:

```
ADMIN_STATE_LOCKED → (主密码) → ADMIN_STATE_UNLOCKED
                                      ↓
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
           ADD_USER流程        DELETE_USER流程    LIST_USERS
```

**添加用户流程**:

```
ADD_STEP_NAME → ADD_STEP_RFID → ADD_STEP_FINGERPRINT
      ↓                                    ↓
      └─────────→ ADD_STEP_PASSWORD → ADD_STEP_CONFIRM → DONE
```

**核心函数**:

| 函数 | 说明 |
|------|------|
| `user_admin_enter()` | 进入管理模式 |
| `user_admin_exit()` | 退出管理模式 |
| `user_admin_start_add_user()` | 开始添加用户 |
| `user_admin_start_delete_user()` | 开始删除用户 |
| `user_admin_on_rfid()` | 录入RFID |
| `user_admin_on_fingerprint()` | 录入指纹 |
| `user_admin_confirm()` | 确认操作 |
| `user_admin_cancel()` | 取消操作 |

## 5. Flash存储布局

### 5.1 地址分配

```
CH32V307 Flash (256KB)
┌────────────────────────────────────────────────────────┐
│ 0x08000000 ─────────────────── Firmware Start          │
│     │                                                  │
│     │  Firmware Code (~24KB)                           │
│     │                                                  │
│ 0x08006000 ─────────────────── Firmware End (approx)   │
│     │                                                  │
│     │  (Reserved/Unused)                               │
│     │                                                  │
│ 0x08040000 ═══════════════════ Access Log Start        │
│     │  Header (256 bytes)                              │
│     │  Record[0..255] × 16 bytes = 4KB                 │
│     │  Total: ~24KB allocated                          │
│ 0x08046000 ═══════════════════ User Database Start     │
│     │  Header (32 bytes)                               │
│     │  User[0..49] × 32 bytes = 1.6KB                  │
│     │  Total: 8KB allocated                            │
│ 0x08048000 ─────────────────── Flash End               │
└────────────────────────────────────────────────────────┘
```

### 5.2 数据格式

**用户数据库头部 (32字节)**:
```
Offset  Size  Field
0x00    4     magic (0x55AA1234)
0x04    2     version
0x06    2     user_count
0x08    2     next_id
0x0A    4     master_password[4]
0x0E    18    reserved
```

**日志头部 (256字节)**:
```
Offset  Size  Field
0x00    4     magic (0xAA55BEEF)
0x04    2     version
0x06    2     total_records
0x08    2     head_index
0x0A    2     tail_index
0x0C    4     sequence_number
0x10    240   reserved
```

## 8. 使用指南

### 8.1 系统启动

系统上电后自动执行以下初始化:

```
1. 硬件初始化 (LED, GPIO, UART, SPI, TIM)
2. 用户数据库初始化 (检查Flash，必要时格式化)
3. 日志模块初始化
4. 认证管理器初始化
5. RFID/指纹模块初始化
6. 启动调度器
```

### 8.2 普通开门流程

**RFID刷卡**:
```
刷卡 → RFID模块检测 → 认证管理器处理 → 查找用户 → 开门/拒绝
                                              ↓
                                         记录日志
```

**指纹验证**:
```
按键13 → 采集指纹 → 搜索匹配 → 认证管理器处理 → 开门/拒绝
                                              ↓
                                         记录日志
```

**密码开门**:
```
按数字键 → 启动密码输入 → 输入4位 → 认证管理器处理 → 开门/拒绝
                                              ↓
                                         记录日志
```

### 8.3 管理模式操作

**进入管理模式**:
```
1. 按 键14                → 启动密码输入
2. 输入主密码:
   键10 键10 键10 键10    → 输入 0000 (默认主密码)
3. 系统验证主密码 → 进入管理菜单
4. 串口输出管理菜单:
   [Admin] ---- Admin Menu ----
   [Admin] 1=Add User  2=Delete User
   [Admin] 3=List Users  4=Change Master Pwd
   [Admin] 15=Exit
```

**添加用户**:
```
1. 按 键1                 → 开始添加用户
2. 按 键16 (确认)         → 确认默认用户名，进入RFID录入
3. 刷RFID卡               → 录入卡号，进入指纹录入
   或 按 键16 (确认)      → 跳过RFID，进入指纹录入
4. 按指纹                 → 录入指纹，进入密码设置
   或 按 键16 (确认)      → 跳过指纹，进入密码设置
5. 输入4位密码             → 设置密码，进入确认步骤
   或 等待超时(10s)        → 跳过密码，进入确认步骤
6. 按 键16 (确认)         → 完成添加，返回管理菜单
```

**删除用户**:
```
1. 按 键2                 → 进入删除模式
2. 按 键1~键9             → 选择用户序号(1=第1个用户, 2=第2个...)
3. 按 键16 (确认)         → 执行删除，返回管理菜单
   或 按 键15 (取消)      → 取消删除
```

**查看用户列表**:
```
1. 按 键3                 → 打印所有用户信息到串口
   输出格式: [序号] ID=用户ID Name=名称 RFID:卡号 FP:指纹ID PWD:Yes/No
2. 按任意键               → 返回管理菜单
```

**修改主密码**:
```
1. 按 键4                 → 开始修改主密码
2. 输入旧密码 (4位)       → 验证旧密码
3. 输入新密码 (4位)       → 设置新密码
4. 再次输入新密码 (4位)   → 确认新密码
```

**退出管理模式**:
```
按 键15 或 键12           → 退出管理模式
双击 键16                → 强制退出管理模式
60秒无操作                → 自动退出管理模式
```

### 8.4 安全机制

**认证失败锁定**:
- 连续5次认证失败后锁定60秒
- 锁定期间所有认证请求被拒绝
- 锁定到期后自动解锁

**管理模式超时**:
- 60秒无操作自动退出管理模式
